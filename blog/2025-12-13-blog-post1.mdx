<h2>求學過程 🏅</h2>
在我大學時期，我第一次接觸到程式語言，並逐漸發現自己對利用程式解決問題的熱情與興趣。從那時開始，我便專注於學習各種程式語言，並將其應用於實際問題中。這些年來，我學習了多種程式語言，包括 C++、C、C#、Python 和 Java，並在學習過程中培養了扎實的程式設計基礎與邏輯思維能力。

<!--truncate-->

在大學的學習過程中，面對著程式設計的挑戰，我學會了如何從零開始，克服困難並逐步掌握一門程式語言。每次遇到不會寫的問題，我便開始尋找問題的根源，並利用搜尋引擎和各種學習資源，快速解決問題，這樣的學習方式讓我學會了如何在有限的時間內找到合適的解決方案。這也讓我在團隊合作中扮演了重要的角色，並且能夠有效地進行問題分析和解決。

在求學的過程中，難免會遇到程式不會寫的問題，這時找答案的能力就格外重要，
在現今搜尋引擎如此發達的年代，我們更應該要培養出找問題的方法，而我也從中學習到找問題的原因以及如何在有限的時間內找到答案，我覺得這點非常重要。另外，在分組報告的部分我常負責製作簡報、文書處理的任務，而成品也獲得許多老師的肯定，從製作簡報中學習如何把資料統整起來，把版面配置跟內容搭配起來，
如此一來，才能讓觀看者迅速地掌握到重點。
我是上大學才第一次接觸到程式語言這個領域，其實對新手的我來說，壓力相當的巨大，要從一開始甚麼都不知道到精通一個程式語言真的很艱難，但是要如何克服這個壓力又是另外一回事了，而我也是挺了這個壓力一路走過來，所以我非常清楚這是多麼困難的一件事情，不過一旦通過了這一關，我認為已經足夠面對接下來的種種考驗了。我不敢說自己真的很擅長很會寫程式，但是基礎的程式架構我還是游刃有餘，因為寫程式需要先釐清一套清楚的演算法，否則寫到一半卡住，就會不知道自己在寫甚麼了。因此，在寫程式之前，先想好自己的演算法是非常重要的。


在我進入研究所後，主要專注於網頁前後端開發的領域。在前端開發方面，我主要使用 React，並且掌握了各種前端技術以確保網頁的流暢度與互動性。在後端方面，我有使用 Node.js 和 Python Flask 開發 RESTful API 的經驗，並且能夠處理各種伺服器與資料庫(MySQL)的相關問題。此外，我也學會了如何利用 Docker 進行容器化操作，並且有使用 AWS 的經驗，包括 EC2、ECS、S3 和 RDS 等服務，這些經驗讓我對雲端運算架構有了深入的了解。

在進行分組報告的過程中，我經常負責製作簡報和文書處理的工作，這不僅讓我學會了如何高效地整理資料，還掌握了簡報設計與內容呈現的技巧。我能夠將複雜的資訊簡化，清晰地呈現給觀眾，並因此獲得了老師和同學的認可，這也進一步增強了我在團隊合作中的影響力。

儘管我在程式設計的道路上起步較晚，但我不斷努力克服困難，並在每一次挑戰中成長。面對複雜的問題，我總是先設計清晰的演算法，並用流程圖或偽代碼理清思路，這樣的習慣讓我能夠更有效率地完成工作並處理各種技術問題。雖然我不敢說自己是各種程式語言的專家，但我能穩定地完成基礎架構設計，並持續精進自己的能力。

這些經歷讓我對未來充滿信心，並激發了我對資工領域更多的探索。我希望能夠參與更多的專案，將我所學的知識與技能應用於解決實際問題，並為這個領域做出貢獻。
